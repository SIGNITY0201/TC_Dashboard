### 프로젝트 통합
- 기존 3개 별도 GitHub repo(대시보드, 견적서, 계약서)를 **1개 repo**로 통합
- 멀티 페이지 구조 적용: `index.html` (대시보드) → `quotation/` (견적서) → `contract/` (계약서)
- 대시보드의 외부 링크를 내부 상대경로(`./quotation/`, `./contract/`)로 변경
- 견적서, 계약서에 **대시보드 복귀 네비게이션** 추가

### 견적서 - Glass Folding 제품 추가
- Glass 카테고리에 **Glass Folding** 품목 신규 추가
- 자재 단가: 350,000원/짝
- 기타 색상 선택 시 +15% 할증 적용
- 시공비 자동 연동 로직:
  - 6짝 미만: 기본 시공비 600,000원 (고정)
  - 6짝 이상: 짝당 100,000원
- 운임/양중비 트럭 카운트에 Glass Folding 반영
- 시공비 테이블 정렬 우선순위에 포함
- PDF 저장 시 한글명 "글라스폴딩" 매핑

### PWA (앱 설치) 지원
- **manifest.json** 추가: 앱 이름, 아이콘, 테마 색상 설정
- **service-worker.js** 추가: 오프라인 캐시 및 네트워크 우선 전략
- **앱 아이콘** 자동 생성 (icon-192.png, icon-512.png)
- 크롬에서 "앱 설치" 버튼으로 바탕화면/홈화면에 독립 앱으로 추가 가능
- iOS Safari에서도 "홈 화면에 추가"로 설치 가능

### 폴더 구조
```

├── icon-512.png            # 앱 아이콘 (512x512)
├── 테라까사-로고.png          # 공유 로고
├── quotation/              # 견적서 프로그램
│   ├── index.html
│   ├── app.js
│   ├── styles.css
│   └── New OASIS 출시 안내문.png
├── contract/               # 계약서 프로그램
│   └── index.html
└── CHANGELOG.md            # 이 파일
```

### 배포 방법
1. GitHub에 새 repo 생성
2. ver1.0 폴더 안의 파일 전부 업로드 (CHANGELOG.md 제외 가능)
3. Settings → Pages → main 브랜치 활성화
4. 단일 URL로 대시보드/견적서/계약서 모두 운영
5. 크롬 주소창 우측 "설치" 아이콘 클릭 → 앱처럼 사용
